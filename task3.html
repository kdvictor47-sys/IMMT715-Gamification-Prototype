Task 3: Accessibility Basics

Learn how to make your HTML more inclusive before starting Level 3.

Web accessibility means building websites that can be used by as many people as possible, including people who use screen readers, keyboard navigation, or other assistive technologies.

Alt text for images

The alt attribute on an <img> tag provides a text description of the image. Screen readers read this out loud if the user can’t see the image.

<img src="cat.jpg" alt="A black cat sitting on a windowsill.">

Good alt text is concise but meaningful. It should describe the content or purpose of the image, not just say "image" or "picture".

Semantic structure

Semantic elements like <nav>, <main>, <header>, and <footer> give structure to your page. Assistive technologies use these landmarks to jump around quickly.

<nav> <ul> <li><a href="#home">Home</a></li> <li><a href="#about">About</a></li> <li><a href="#contact">Contact</a></li> </ul> </nav>

Links and buttons with clear text

Link and button text should clearly say what they do. Avoid vague labels like "Click here". Instead, use text like "Read more about pricing" or "Submit form".

Keyboard navigation

Many users navigate with a keyboard using the Tab key to move between focusable elements. Make sure your page can be used without a mouse: links and buttons should be focusable and visible.
